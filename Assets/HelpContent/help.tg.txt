## 1) Ин чист ва барои чӣ
DevProjex контексти лоиҳаро зуд омода мекунад: сохтори ҷузвдонҳо/файлҳоро нишон медиҳад, имкон медиҳад элементҳоро бо қуттӣ интихоб кунед ва дарахт ё мундариҷаи файлҳоро ба буфер нусха бардоред. Барои code review, дастгирӣ, омӯзиш ва додани сохтори лоиҳа ба ёрдамчии AI қулай аст.

Муҳим: барнома танҳо дар режими хондан кор мекунад ва файлҳоро тағйир намедиҳад.

## 2) Дарахти лоиҳа
* Иконкаҳои навъи файл/ҷузвдон.
* Қуттиҳои интихоб дар гиреҳҳо.
* Кушодан/бастан.

Қуттиҳо: интихоб ба наслҳо татбиқ мешавад; ҳолати падар худкор ҳисоб мешавад.

Тугмаҳои зуд:
* Ctrl+E — кушодани ҳама
* Ctrl+W — бастани ҳама (реша кушода мемонад)

## 3) Файл
* Кушодани ҷузвдон… — кушодани ҷузвдони реша (Ctrl+O)
* Навсозӣ — бозхондан ва бозсозии дарахт (F5)
* Содирот ба файл:
  * Содироти дарахт
  * Содироти мундариҷа
  * Содироти дарахт ва мундариҷа

Қоидаҳои формат:
* Содироти дарахт:
  * ASCII -> танҳо `.txt`.
  * JSON -> пешфарз `.json`, дар равзанаи нигоҳдорӣ `.json` ва `.txt` дастрасанд.
* Содироти мундариҷа — ҳамеша формати матнии мундариҷа, файл `.txt` (тугмаи ASCII/JSON таъсир надорад).
* Содироти дарахт ва мундариҷа — ҳамеша файл `.txt`; тугмаи ASCII/JSON танҳо ба формати блоки дарахт дар натиҷа таъсир мерасонад.

Файлҳои содирот бо UTF-8 (бе BOM) навишта мешаванд.

### Drag & Drop
Drag & Drop дар майдони оғоз (Drop Zone) кор мекунад, вақте ки лоиҳа бор нашудааст.

Қоидаҳо:
* метавонед ҷузвдонро кашида партоед;
* метавонед файлро кашида партоед — дар ин ҳолат ҷузвдони он кушода мешавад;
* агар чанд объект партофта шавад, аввалин роҳи маҳаллии мувофиқ истифода мешавад;
* агар роҳ дастнорас бошад ё вуҷуд надошта бошад, ҳамон санҷиши дастрасӣ мисли «Кушодани ҷузвдон…» истифода мешавад;
* объектҳои дастгиринашаванда ё ғайримаҳаллӣ нодида гирифта мешаванд.

## 4) Git
Менюи Git танҳо барои лоиҳаҳое дастрас аст, ки тавассути DevProjex клон шудаанд (Git Clone). Барнома ба файлҳои шумо даст намерасонад — танҳо бо кеш кор мекунад.

Кеш дар ҷузвдони муваққатии система нигоҳ дошта мешавад. Дар Windows одатан:
`%TEMP%\DevProjex\RepoCache`

* Шоха — рӯйхати шохаҳои дурдастро нишон медиҳад; ҳангоми иваз кардан дарахт ба шохаи интихобшуда нав мешавад. Агар шоха аллакай маҳаллӣ бошад, ивазкунӣ бе шабака (бе навсозӣ) сурат мегирад. Агар набошад, он тавассути fetch бор карда шуда, маҳаллӣ сохта мешавад.
* Гирифтани навсозиҳо — танҳо шохаи ҷориро нав мекунад: fetch ва reset ба origin/<шоха>.

## 5) Нусха
### Тугмаи формати дарахт (ASCII/JSON)
Ин тугма формати намоиши дарахтро барои амалиётҳое муайян мекунад, ки дар натиҷа дарахт иштирок мекунад.

* ASCII — дарахти матнӣ бо шохаҳо.
* JSON — дарахти сохторӣ дар JSON.

Тугма ба инҳо таъсир мерасонад:
* Нусхаи дарахт
* Нусхаи дарахт + мундариҷа
* Содироти дарахт
* Содироти дарахт ва мундариҷа

Тугма ба инҳо таъсир намерасонад:
* Нусхаи мундариҷа
* Содироти мундариҷа

### Нусхаи дарахт
Дарахти лоиҳаро нусха мебардорад.
Агар гиреҳҳои интихобшуда бошанд, дарахт аз рӯйи интихоб сохта мешавад; агар интихоб холӣ ё номувофиқ бошад — дарахти пурра нусха мешавад.
Формати натиҷа бо тугмаи ASCII/JSON муайян мешавад.
Тугмаҳо: Ctrl+Shift+C ва Ctrl+Alt+C.

### Нусхаи мундариҷа
Мундариҷаи файлҳоро нусха мекунад:
* агар файлҳо интихоб шуда бошанд — танҳо онҳо; вагарна — ҳамаи файлҳои лоиҳа;
* файлҳои ғайриматнӣ нодида гирифта мешаванд (аз ҷумла бинарӣ);
* барои файлҳои матнии холӣ роҳ ва сатри `[No Content, 0 bytes]` бароварда мешавад;
* барои файлҳое, ки танҳо аз фосилаҳо/гузариши сатрҳо иборатанд, роҳ ва сатри `[Whitespace, N bytes]` бароварда мешавад.
Тугма: Ctrl+Alt+V.

### Нусхаи дарахт + мундариҷа
Дарахт ва мундариҷаро якҷо мекунад:
* дарахт аз интихоб; агар интихоб холӣ ё номувофиқ бошад — дарахти пурра;
* мундариҷа мувофиқи қоидаҳои «Нусхаи мундариҷа»;
* формати блоки дарахт бо тугмаи ASCII/JSON муайян мешавад.
Тугма: Ctrl+Shift+V.

### Намоиши JSON
Дар режими JSON сохтори фишурдаи дарахт истифода мешавад.

Дарахт (Нусхаи дарахт / Содироти дарахт):
* `rootPath` — роҳи мутлақи реша.
* `root` — гиреҳи реша.
* майдонҳои гиреҳ: `name`, `path`, `dirs`, `files`, `accessDenied` (ҳангоми зарурат).

Дарахт + мундариҷа:
* JSON ба блоки дарахт татбиқ мешавад;
* блоки мундариҷа матнӣ мемонад;
* содирот дар `.txt` захира мешавад.

## 6) Реҷаи пешнамоиш
Бо тугмаи нишонаи чашм дар панели боло ё бо Ctrl+B кушода мешавад.

Натиҷаи ниҳоиро дар реҷаи танҳо-хондан нишон медиҳад: бе нусхабардорӣ ба буфер ва бе содирот ба файл.

Реҷаҳои дастрас:
* Дарахт
* Мундариҷа
* Дарахт ва мундариҷа

Ҳамон қоидаҳое татбиқ мешаванд, ки барои нусхабардорӣ ва содирот:
* гиреҳҳои интихобшуда ба ҳисоб гирифта мешаванд;
* агар интихоб холӣ бошад, маҷмӯи пурраи додаҳо истифода мешавад;
* қоидаҳои ҷории нодидагирӣ ва параметрҳои интихоб татбиқ мешаванд.

Ҳангоми боз будани пешнамоиш, ҷустуҷӯ ва филтр дастрас нестанд.
Дар танзимот танҳо тағйири шрифт дастрас аст.
Бастан: тугмаи X ё пахши дубораи нишонаи чашм.

## 7) Намоиш
### Масштаб
* Ctrl++ / Ctrl+= — калон кардан
* Ctrl+- — хурд кардан
* Ctrl+0 — баргардондан
* Ctrl + чархи муш — зум нарм

Диапазон: 6–28, баргардондан — 12.

### Режими ихчам
Интерфейсро зичтар мекунад: фосилаҳо дар дарахт ва танзимот хурдтар мешаванд.

### Аниматсияи дарахт
Ҳангоми бурдани курсор ба сатрҳои дарахт аниматсияи ҳаракати нармро фаъол мекунад.

## 8) Ҷустуҷӯ дар дарахт
Панели ҷустуҷӯро мекушояд, мувофиқатҳоро равшан мекунад ва навигатсия медиҳад.
* Ctrl+F — нишон/пинҳон
* Enter — натиҷаи навбатӣ
* Shift+Enter — қаблӣ
* Esc — пӯшидан

Ҷустуҷӯ танҳо шохаҳои заруриро мекушояд; ҳангоми тозакунии дархост дарахт ҷамъ мешавад (ғайр аз реша).

## 9) Филтр бо ном
Танҳо гиреҳҳои мувофиқ ва падарони онҳоро нишон медиҳад. Навсозӣ бо таъхир.
* Ctrl+Shift+N — филтрро нишон/пинҳон
* Esc — пӯшидани филтр

## 10) Танзимот
Дар тарафи рост рӯйхатҳо:
* Шрифти дарахт — бо Истифодаи танзимот татбиқ мешавад
* Нодида гирифтан
* Навъҳои файл
* Ҷузвдонҳои сатҳи боло

Дар ҳар рӯйхат тугмаи Ҳама (ҳамаро фаъол/хомӯш) мавҷуд аст.

Эзоҳ дар бораи «Навъҳои файл»:
* дар ин рӯйхат танҳо файлҳои дорои васеъшавӣ нишон дода мешаванд;
* файлҳои бе васеъшавӣ ба ин рӯйхат дохил намешаванд ва тавассути банди алоҳида дар «Нодидагирӣ» идора мешаванд.

Ctrl+P — панелро нишон/пинҳон.

## 11) Нодидагирӣ
Рӯйхати «Нодидагирӣ» аз рӯи роҳи ҷорӣ ва ҷузвдонҳои сатҳи болоии интихобшуда ба таври худкор сохта мешавад.

Тартиби пунктҳо:
* Smart ignore (агар дастрас бошад)
* Истифодаи .gitignore (агар ҳадди ақал як `.gitignore` ёфт шавад)
* Ҷузвдонҳои пинҳон
* Файлҳои пинҳон
* Ҷузвдонҳо бо нуқта дар оғоз
* Файлҳо бо нуқта дар оғоз
* Файлҳои бе васеъшавӣ (танҳо агар чунин файлҳо ёфт шаванд, нишон дода мешавад)

Хатти зери «Истифодаи .gitignore» қоидаҳои Git/Smart-ро аз қоидаҳои асосии намоиш ҷудо мекунад.

### 11.1 Минтақаи татбиқи қоидаҳо
Барнома минтақаҳои лоиҳа (scope)-ро месозад, ки қоидаҳои нодидагирӣ дар онҳо татбиқ мешаванд:
* реша (агар дар реша `.gitignore` ё нишонаи лоиҳа бошад)
* ҷузвдонҳои сатҳи аввал
* зерҷузвдонҳои дохилии лоиҳа

Амиқии ҷустуҷӯи минтақаҳои дохилӣ собит аст: 2 сатҳ.

### 11.2 «Истифодаи .gitignore»
* Фаъол: қоидаҳои `.gitignore` дар минтақаҳои ёфтшуда татбиқ мешаванд.
* Ғайрифаъол: қоидаҳои `.gitignore` пурра хомӯш мешаванд.

Дар ҷузвдони бисёрлоиҳаӣ, `.gitignore`-и як лоиҳа ба лоиҳаҳои ҳамсоя берун аз минтақаи худ татбиқ намешавад.

### 11.3 «Smart ignore»
* Танҳо вақте нишон дода мешавад, ки воқеан зарур бошад.
* Фаъол: артефактҳои маъмулии build/tool иловатан пинҳон мешаванд.
* Ғайрифаъол: smart-қоидаҳо татбиқ намешаванд.

### 11.4 Қоидаҳои асосӣ
Пунктҳо:
* Ҷузвдонҳои пинҳон
* Файлҳои пинҳон
* Ҷузвдонҳо бо нуқта дар оғоз
* Файлҳо бо нуқта дар оғоз

Ин қоидаҳо аз `.gitignore` ва Smart Ignore мустақиланд ва танҳо аз рӯи гузинаҳои худ кор мекунанд.

### 11.5 Коркарди дохилӣ: ҳолатҳои маъмул
1. Як лоиҳа, `.gitignore` ҳаст.
Одатан танҳо «Истифодаи .gitignore» нишон дода мешавад.
Ҳангоми фаъол будан, `.gitignore` ва Smart Ignore якҷоя кор мекунанд.
Ҳангоми ғайрифаъол будан, ҳарду хомӯш мешаванд.

2. Як лоиҳа, `.gitignore` нест.
«Истифодаи .gitignore» нишон дода намешавад.
«Smart ignore» танҳо вақте нишон дода мешавад, ки мувофиқ бошад.

3. Workspace-и бисёрлоиҳаӣ: қисми лоиҳаҳо `.gitignore` доранд, қисми дигар надоранд.
«Истифодаи .gitignore» ва «Smart ignore» нишон дода мешаванд.
`.gitignore` дар минтақаҳои худ амал мекунад.
Smart Ignore (агар фаъол бошад) филтркуниро дар минтақаҳои ёфтшуда пурра мекунад.

4. Workspace-и бисёрлоиҳаӣ: ҳамаи лоиҳаҳо `.gitignore` доранд.
«Истифодаи .gitignore» нишон дода мешавад.
«Smart ignore» ҳамчун пунктҳои ҷудогона нишон дода намешавад.

5. Ҷузвдон бе қоидаҳои мувофиқ.
Агар `.gitignore` ёфт нашавад ва Smart Ignore мувофиқ набошад, ин пунктҳо нишон дода намешаванд.
Танҳо қоидаҳои асосии намоиш мемонанд.

### 11.6 Файлҳои бе васеъшавӣ
Банди «Файлҳои бе васеъшавӣ» намоиши файлҳои бе васеъшавиро идора мекунад (масалан: `Dockerfile`, `Makefile`, `README`).

Қоидаҳо:
* ин банд танҳо вақте нишон дода мешавад, ки чунин файлҳо воқеан дар маҷмӯи ҷории маълумот вуҷуд дошта бошанд;
* ҳангоми фаъол будани банд, файлҳои бе васеъшавӣ аз дарахт ва аз амалиётҳои нусха/содирот хориҷ мешаванд;
* ҳангоми ғайрифаъол будани банд, файлҳои бе васеъшавӣ ба натиҷа дохил мешаванд;
* рӯйхати «Навъҳои файл» ба ин механизм таъсир намерасонад.
## 12) Дарахт ва метрикаҳо чӣ тавр ҳисоб мешаванд
Натиҷа вобаста аст ба:
* интихобшуда ҷузвдонҳои сатҳи боло
* интихобшуда навъҳои файл
* қоидаҳои нодида гирифтан
* филтр фаъол
* гиреҳҳои бо checkbox интихобшуда дар дарахт (барои амалиётҳои вобаста ба интихоб)

Агар дастрасӣ набошад, гиреҳ бо Дастрасӣ нест нишон дода мешавад.

### 12.1 Ҳисобкунакҳои сатри ҳолат чиро нишон медиҳанд
Дар сатри ҳолат ду блок ҳаст:
* Дарахт — метрикаҳои барориши матнии дарахт;
* Мундариҷа — метрикаҳои барориши мундариҷаи файлҳо.

Формат:
* Сатрҳо
* Аломатҳо
* ~Токенҳо (тахминӣ)

Қиматҳои калон дар формати кӯтоҳ (K/M) нишон дода мешаванд.

Муҳим:
* «Сатрҳо» дар блоки «Дарахт» — шумораи сатрҳои намоиши ниҳоии дарахт аст, на шумораи файлҳо дар диск;
* ба ин арзиш ҷузвдонҳо, файлҳо ва унсурҳои хизматии дарахт (масалан, гиреҳҳои дастнорас) дохил мешаванд.

### 12.2 Метрикаҳо чӣ тавр ҳисоб мешаванд
Метрикаҳои дарахт:
* аз матни воқеии дарахт ҳисоб мешаванд, ки аз ҳамон pipeline-и содирот сохта мешавад;
* формати ҷории ASCII/JSON-ро ба назар мегиранд.

Метрикаҳои мундариҷа:
* бо ҳамон қоидаҳои нусха/содироти мундариҷа ҳисоб мешаванд;
* файлҳои бинарӣ дохил намешаванд;
* файлҳои холӣ ва танҳо фосиладор бо маркерҳои хизматӣ ҳисоб мешаванд (`[No Content, 0 bytes]`, `[Whitespace, N bytes]`).

### 12.3 Ҳисобкунакҳо кай нав мешаванд
Бозҳисобкунӣ иҷро мешавад:
* пас аз кушодан/навсозии лоиҳа;
* пас аз татбиқи танзимот (ignore, навъҳои файл, ҷузвдонҳои реша);
* ҳангоми тағйири checkbox дар дарахт (бо debounce);
* ҳангоми иваз кардани ASCII/JSON (барои амалиётҳое, ки дарахтро истифода мебаранд).
* агар ҳисобкунии фонӣ бекор шавад, ҳисобкунакҳо то бозҳисобкунии пурраи навбатӣ ба 0 гузошта мешаванд.

## 13) Иконкаҳо
Тартиби интихоб:
1) навъи гиреҳ (ҷузвдон/файл)
2) номи дақиқ (агар қоида бошад)
3) васеъшавӣ
4) дар акси ҳол — номаълум

## 14) Мавзӯъ ва эффектҳо
Popover‑и мавзӯъ имкон медиҳад:
* мавзӯи равшан/торик
* Шаффоф / Mica / Acrylic
* параметрҳои интенсивият, blur, контраст ва сарҳад

Бандҳои бо [Beta] метавонанд байни версияҳо тағйир ёбанд.

## 15) Ёрӣ → Дар бораи барнома
Popover бо маълумот ва истиноди репозиторийро мекушояд.
Барқарор кардани танзимот (Ёрӣ → Барқарор кардани танзимот) танзимоти мавзӯъ ва эффектҳоро ба қиматҳои ибтидоӣ бармегардонад.
* Esc — пӯшидани popover

## 16) Забонҳо
Забон аз менюи Забон интихоб мешавад.

## 17) Progress-бар ва бекоркунӣ
Дар қисми марказии сатри ҳолат амалиёти фаъол ва прогресс нишон дода мешавад.

Режимҳои прогресс:
* Indeterminate — бидуни фоиз (давомнокӣ номаълум аст).
* Determinate — бо фоиз (0–100).

Рафтор:
* ҳангоми ҳисобкунии фонӣ фоизи прогресс нишон дода мешавад;
* барои қисми амалиётҳои Git фоиз танҳо вақте нишон дода мешавад, ки манбаи прогресс онро диҳад;
* барои амалиётҳои хеле зуд прогресс метавонад кӯтоҳ намоён шавад.

Тугмаи X (бекоркунӣ) дар сатри ҳолат:
* барои амалиётҳое, ки бекоркуниро дастгирӣ мекунанд, best-effort бекоркунӣ иҷро мекунад;
* ҳисобкунии фонии маълумотро бозмедорад;
* барои амалиётҳои бе дастгирии бекоркунӣ индикацияро дар сатри ҳолат мебандад, вале худи амалиёт метавонад дар фон анҷом ёбад.

## 18) Эзоҳҳо
* Пешфарз мавзӯи торик ва эффектҳои шаффофият фаъол аст (агар дастгирӣ шавад).
* Ҳангоми нокифоя будани иҷозат, барнома метавонад бозоғозӣ бо ҳуқуқи баланд пешниҳод кунад; агар рад шавад — огоҳӣ нишон дода мешавад.
